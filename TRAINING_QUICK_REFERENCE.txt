THE COMPLETE WORKFLOW
═════════════════════════════════════════════════════════════════════════════════

Week 0 (Friday/Saturday):
  📹 Record 5-10 FRC matches at your event
     • Side view of field (best for bumpers)
     • 15-40 minutes total footage
     • Good lighting and steady camera

Week 1 - Monday evening:
  🖼️ Extract frames from videos (5 minutes)
     python train_detector.py extract
     → Creates 1000+ JPG frames in videos/frames/
  
  🏷️ Annotate robots on Roboflow (1-2 hours)
     • Upload images to Roboflow
     • Draw bounding boxes around robots
     • Label as class "robot"
     • Download dataset in YOLOv8 format

Week 1 - Tuesday evening:
  🤖 Train model (30 min - 2 hours)
     python train_detector.py train
     → Trains YOLOv8 on YOUR annotated data
     → Saves to: models/yolov8_robots.pt
  
  ✅ Evaluate accuracy (5 minutes)
     python train_detector.py evaluate
     → Shows mAP, precision, recall
  
  📊 Test on video (10 minutes)
     python train_detector.py test videos/raw/week0_match1.mp4
     → Verify detection working on actual footage

Week 1 - Wednesday:
  🔧 Integrate into app (10 minutes)
     • Update robot_detector.py with real inference
     • Point to models/yolov8_robots.pt
     • Test on live camera
  
  🚀 READY FOR DEPLOYMENT!

WHAT HAPPENS AT EACH STEP
═════════════════════════════════════════════════════════════════════════════════

Frame Extraction:
  INPUT: videos/raw/*.mp4
  PROCESS: Read frames, save every 5th frame
  OUTPUT: videos/frames/*.jpg (1000+ images)
  TIME: 5 minutes

Annotation (on Roboflow):
  INPUT: videos/frames/*.jpg
  PROCESS: Draw bounding boxes, label "robot"
  OUTPUT: dataset/ folder with YOLO format
  TIME: 1-2 hours (manual work)

Training:
  INPUT: dataset/images/train + labels
  PROCESS: Train YOLOv8 model on your data
  OUTPUT: models/yolov8_robots.pt
  TIME: 30 min (GPU) - 2 hours (CPU)

Evaluation:
  INPUT: Trained model + validation set
  PROCESS: Measure accuracy metrics
  OUTPUT: mAP, precision, recall scores
  TIME: 5 minutes

Testing:
  INPUT: Video file + trained model
  PROCESS: Run inference, draw boxes
  OUTPUT: Annotated video with detections
  TIME: 10 minutes

Integration:
  INPUT: Trained model
  PROCESS: Load in RobotDetector
  OUTPUT: Working app with detection + tracking
  TIME: 10 minutes

TIPS FOR BEST RESULTS
═════════════════════════════════════════════════════════════════════════════════

Recording:
  ✓ Film at least 5 different matches
  ✓ Vary: times, lighting, team compositions
  ✓ Use tripod for steady footage
  ✓ Position for bumper visibility

Annotation:
  ✓ Make bounding boxes TIGHT
  ✓ Be consistent in labeling
  ✓ Include partial robots at edges
  ✓ Go faster after first 50 images

Training:
  ✓ Start with 300-500 annotated images
  ✓ Use GPU if available (much faster)
  ✓ Monitor metrics, they should improve
  ✓ Early stopping is normal (good sign)

Improvement:
  ✓ Record more matches throughout season
  ✓ Retrain every 2-3 weeks
  ✓ Model gets better with more data
  ✓ By playoffs: excellent accuracy


TROUBLESHOOTING QUICK GUIDE
═════════════════════════════════════════════════════════════════════════════════

Problem: Can't extract frames?
  → Check videos/raw/ exists
  → Check video files play in VLC
  → Check free disk space

Problem: Training very slow?
  → Using GPU? Check CUDA setup
  → Reduce batch size: 16 → 8
  → Reduce epochs: 100 → 50

Problem: Low accuracy?
  → Need 300+ annotated images (minimum)
  → Check annotation quality
  → More diverse footage
  → Train longer
